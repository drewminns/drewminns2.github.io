Colours

Modern browser can benefit from the use of a broad spectrum of colours to style it’s elements. Typically colours are defined through hexidecimal values, commonly known as ‘hex values’.

Hexidecimal

color: #000000;

Hex Values are read by a browser as a code to colours. It is treated as Red/Green/Blue spectrum with each colour given a two digit group of characters. The first and secret characters represent the red value, the third and forth characters represent the green value, the fifth and sixth characters represent the blue value. The characters range from a-f and 0-9 to create colour combinations.

// Red
color: #ff0000;
// Green
color: #00ff00;
// Blue
color: #0000ff;

RGBA

color: rgba(255,255,255,0.8);

While hex values are great for creating solid colours, what if you need a colour that has some transparency. Enter RGBA, meaning Red Blue Green Alpha. Instead of using hex codes for each value, the colours are defined by a range from 0-255. The final decimal value indicates the alpha level of the colour.

// Red 50% Alpha
color: rgba(255,0,0,0.5);
// Green 20% Alpha
color: rgba(0,255,0,0.2);
// Blue 95% Alpha
color: rgba(0,0,255,0.95);

Backgrounds

Instead of using colours for backgrounds, images can be incorporated in a number of ways in your CSS

Background-Image

Using the background image value can allow an image to be set as your background.

background-image: url('background.jpg');

By default, this image will be the size of the image, will repeat across the page, start in the top left corner, and will scroll with the page.

Background-Repeat

If you do not want the background to repeat across the page, or only want the background to repeat a certain direction or not repeat at all, you can use the background-repeat property.

// Only repeat horizontally
background-repeat: repeat-x;
// Only repeat vertically
background-repeat: repeat-y
// Do not repeat
background-repeat: no-repeat;

Background-Position

You can adjust where the background will begin on your site using the background-position property. This is particularly useful in use with the background-repeat property, if you want an image certained in the page. The values that it takes, work on the x and y dimensions of a page or by simply defining generic positioning values.

// Center Background Image vertically and horizontally
background-position: center center;
// Background Image at top left corner
background-position: left top;
//Background Image 200px from left and 500px from top
background-position: 200px 500px;
//Background Image 150px from left and at the bottom
background-position: 150px bottom;

Background-Attachment

By default, all background images will scroll with the page. You can change this functionality by defining a fixed value for the background-attachment property.

//Fixed Position Background
background-attachment: fixed;

Background Size

By default, background images will use the size that they are saved at, but you can define the size that they are presented on a page. These images can be defined using width and height values in pixels or percentages.

// Background Size 800px wide and 200px tall
background-size: 800px 200px;
// Background Size 100% wide and 300px tall
background-size: 100% 300px;
// Background full size of screen
background-size: cover;

Box Model

One of the most misunderstood things about a web page is how an object on the object reacts to dimensions given to it. Using border and padding will affect the dimensions of the object, often in ways, that are undesirable.

Margin is used to create an area around the object. This will affect the layout and positioning of objects around it.

Borders are used to define a border around an object. The width of the border, will affect the positioning and layout of objects around it.

Padding is used to create spacing around the content in an object. The value of the padding will affect the position and layout of objects around it.

Fixing the box model problem.

Padding is one of these things that was not developed well within CSS. Padding should not affect the width and height of the object and not move the content around it. Instead, it should give spacing inside the object.

To fix this problem, the box-sizing property can be used to have padding be used in a more effective way.

Use the following code in your CSS to fix the box-model. The asterisk is used to apply it to all objects on the page.

* {
    box-sizing: border-box;
}

Fixed Vs Fluid Layout

Typically, webpages use pixel values to define the dimensions of an object. While this is great for building a solid page, this can make building a page for different devices, very difficult.

Enter flexible layouts.

Using percentage values can ensure your page reacts to the size of the browser. On smaller screens, fixed pixel values will make an object be too large. Percentage based values are based on 0%-100% and relative to it’s encapsulating object. Using a max-width property on the container object will allow the object to have a flexible width.

Fixed Layout with full header, section and aside next to each other

// HTML
<div id="container">
    <header>Header Content</header>
    <section>Section Content</section>
    <aside>Aside Content</aside>
</div>
// CSS
#container {
    width: 1080px;
    margin: 0 auto;
}
header {
    width: 1080px;
}
section {
    float: left;
    width: 740px;
}
aside {
    float: left;
    width: 340px;
}
