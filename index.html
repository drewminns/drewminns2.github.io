Week 9 - jQuery

jQuery

Download In-Class Files https://github.com/drewminns/webd-starter

Ok, first things first.

Javascript is a programming language. jQuery is a library that uses Javascript to make programming user actions on a webpage easier. jQuery was developed to remove some of the more strenuous activities that come with selecting objects on a webpage and interacting, animating and more.

DOM

The Document Object Model is the framework of all elements on a webpage. This includes all objects and elements we place on the page. The web inspector in your browser is the best way to explore the DOM.

CDN VS Hosted

In order to use jQuery, we must import the library into our page. Because Javascript is built into every browser, we simply import the library via a script tag, that hooks into the existing Javascript system.

http://jquery.com/ hosts all the documentation needed and allows you to download versions of jQuery to include in your site heirarchy.

Another option of including jQuery in your page, is using a CDN (Content Delivery Network) to serve the file to your browser. Using a CDN ensures that even if you forget to upload jQuery, it will be linked to your site.

<script src="http://code.jquery.com/jquery-1.11.0.min.js"></script>

Version 1 vs 2.0

jQuery 2.0 was recently released and is the next logical iteration of jQuery to use. While we do our best in this industry to stay up to date, 2.0 does not support IE 6/7/8, which means that we would be leaving out a chunk of our users. That’s why it is best to currently stick with a 1.0 version, particularly 1.11.

Document Ready

To ensure that our page has all functionality for a user to use, we have to ensure that the page is entirely loaded before applying any jQuery animations. To do that, we use a function to check that the DOM is ready. Write this function once and insert all jQuery inside.

$(document).ready(function() {
    // Put jQuery in here
});

Without ensuring that the document is loaded, we risk our javascript not being loaded and functionality being lost. It is imperative that we use the above script at all times when using jQuery.

One of the easiest ways to check that jQuery is working is to put a console.log(“It works!”); inside of the document ready function.

$(document).ready(function() {
    console.log("It works!");
});

With this, we can then use the console in our browser inspector to view the message.

Selecting Objects

In vanilla Javascript, selecting an object was a convuluted task.

//HTML
<div id="myObject"></div>
//JS
var x=document.getElementById("myObject");

From there, we would have to use the variable x to apply animations. With jQuery we use the character $ to instantiate a jQuery command and selecting an object.

//HTML
<div id="myObject"></div>
$(document).ready(function() {
    $("#myObject").someFunction();
});

Functions

jQuery has built in functions that we can utilize to make changes. They are all listed within the API documentation at http://api.jquery.com/. They can be changed together and applied to an object as often as wanted.

$("h3").slideToggle().css("height", "300px").hide();

Changing CSS

Once we have an object selected we can easily apply functions to it that are apart of the jQuery library.

One of those handy functions is css. Using the css function, we can easily change the CSS attributes of an object.

$("h1").css("color", "#fff");

Because the jQuery is applied after the document is loaded, the changes will overwrite anything within the stylesheet.

Adding Classes

Another great usage of jQuery is adding classes to objects, which then have styles applied to them.

//HTML
<div id="box"></div>

// JS
$("#box").addClass("blueBox");
//Outputs
<div id="box" class="blueBox"></div>

Inserting text or HTML to an object

Another group of functions that can be utilized is text() and html(). Using these functions, you can insert content inside of a selected object. With text(), you can insert simple plain text into an object. With the html() function, you can insert html inside of the selected element.

//HTML
<div id="one"></div>
<div id="two"></div>
//JS
$("#one").text("New Content");
$("#two").html("<h2>Look at me!</h2>");
//Outputs
<div id="one">New Content</div>
<div id="two"><h2>Look at me!</h2></div>

Allowing with inserting content inside of an object, we can use .prepend() and .append() to specify where in the object we want to insert the content.

append();

//HTML
<div id="one">
    Hello
</div>
<div id="two">
    Hello
</div>
//JS
$("#one").append("<p>Goodbye</p>");
//Output
<div id="one">
    Hello
    <p>Goodbye</p>
</div>
<div id="two">
    Hello
</div>

prepend();

//HTML
<div id="one">
    Hello
</div>
<div id="two">
    Hello
</div>
//JS
$("#one").append("<p>Goodbye</p>");
$("#two").prepend("<p>Goodbye</p>");
//Output
<div id="one">
    Hello
    <p>Goodbye</p>
</div>
<div id="two">
    <p>Goodbye</p>
    Hello
</div>

Event Handlers

jQuery comes with a wide array of tools used to interact with an object. These handlers bind to the Javascript events that occur in a browser.

Events pertaining to a users mouse actions include click, hover, mousedown, mouseup, mouseleave and more. For the full list, check out http://api.jquery.com/category/events/mouse-events/

For events pertaining to keyboard actions, read http://api.jquery.com/category/events/keyboard-events/

There are many more that can be utilized with the browser itself, a form or a document loading. http://api.jquery.com/category/events/

Every action needs a reaction

For every event handler used, you typically would need to include some to happen. For example, click an object and turn another object green. To do this, we select an object, attach an event handler to it, instatiate a function and insert the reaction within. See below

//CSS
#blue {
    color: #0000ff;
}
//HTML
<h1>Click</h1>
<div id="blue">Hello</div>
//JS
// Select object and apply event handler
$("h1").click(function () {
    // Apply change to another object
    $("#blue").css("color", "#00ff00");
});

The above example would work like this. When the h1 element is clicked, the div with the ID of blue, would have it’s text colour changed to green (#00ff00).

Apply to itself

Now, what if we didn’t want the click to apply a change to another element, we wanted the click to change itself. We can simply use this as the selector.

//HTML
<h1>Click</h1>
<div id="blue">Hello</div>
//JS
// Select object and apply event handler
$("h1").click(function () {
    // Apply change to another object
    $(this).css("color", "#00ff00");
});

Note that this is not inside quotes, because it is a root object. The output would be that the h1 would change colour once it is clicked.

Making our own functions

We can write our own functions to reuse throughout a page, much like how functions are created in regular Javascript. Be sure to include return this to return the change. The use of created functions are imperative to not repeating code over and over. Write it once, and call it as many times as needed.

// Instantiate a function and name it greenify
$.fn.greenify = function() {
  // Apply CSS changes
  this.css( "color", "green" );
  // Return the changes
  return this;
};
// Call greenify function on object
$("h1").greenify();

Animation

Yes, animation. The coolest part of it all. Goodbye Flash, hello Javascript. The way it should be. Animations fall in to the category of Effects in jQuery http://api.jquery.com/category/effects/. With animations, you can fade objects, hide, show, slide, etc. These work by manipulating the dimensions and opacity of HTML elements. They are best used in conjunction with an event handler.

// Using the generic animate effect, we can pass a css rule in to be manipulated on an event. This being on hover, we will change the font size to 48px
$("h1#block").hover(function() {
    $(this).animate({ "font-size":"48px"}, 1000);
}

Oh, that number at the end? That’s the duration. It determines how long the animation will run. A number can be passed through that represents milliseconds. You can also use strings such as Fast and Slow to represent 200 and 600 milliseconds respectively.

Week 8 - Best Practices

Best Practices or: Why we do the things we do

So websites are built with code and images, and ultimately your code could look however you want it to, and your website would still work. However, what happens when you go back to edit your code and there’s no rhyme or reason to how it looks. Now, think of yourself as someone who’s working on a project that someone else started. That person left their code a mess, and it’s not part of your job, and quoted hours, cleaning it up.

It’s important that we maintain standards within our code, not only to help ourselves and others, but there are significant performance and debugging issues when we don’t follow standards.

Below are some common standards followed throughout the Industry

HTML

Syntax

  • Use tabs instead of spaces to format your code. Specifically, use 2 tabs.
    • In Sublime, go to View -> Indentation and set Tab-Width to 2 and always use tabs.
  • Indent nested elements by using tabs

  • Use double quotes on your attributes

      <!DOCTYPE html>
  <html>
    <head>
      <title>Page title</title>
    </head>
    <body>
      <img src="images/company-logo.png" alt="Company">
      <h1 class="hello-world">Hello, world!</h1>
    </body>
  </html>

Practicality

Use the least amount of markup at all times while maintaning simplicity. Avoid using unnecesary elements that will require extra CSS

<!-- Bad -->
<div class="profile">
    <img src="#">
</div>

<!-- Best -->
<img src="#" class="profile">

Attribute Order

Order your attributes accordingly to help with debugging your code.

  • class
  • id
  • name
  • data-*
  • src, for, type, href
  • title, alt

CSS

Syntax

  • Use tabs instead of spacing
  • When grouping selectors, keep each selector to a single line
  • Include one space before the opening curly brace
  • Put closing brackets on a new line
  • Include one space after each colon for each declaration
  • Each CSS rule should appear on it’s own line
  • Lowercase each hex value
  • Use shorthand values where necessary
  • Indent the CSS rules
    /* Bad */
    .box1, .box2, .box3, .box4{
    color:#000000; background-color:#ffffff;}
    /* Good */
    .box1,
    .box2,
    .box3,
    .box4 {
        color: #000000;
        background-color: #ffffff;
    }

Declaration Order

CSS Rules should be declared in a certain order in order to not group certain aspects together

  • Position
  • Box Model
  • Typographic
  • Visual
    .declaration-order {
          /* Positioning */
          position: absolute;
          top: 0;
          right: 0;
          bottom: 0;
          left: 0;
          z-index: 100;

          /* Box-model */
          display: block;
          float: right;
          width: 100px;
          height: 100px;

          /* Typography */
          font: normal 13px "Helvetica Neue", sans-serif;
          line-height: 1.5;
          color: #333;
          text-align: center;

          /* Visual */
          background-color: #f5f5f5;
          border: 1px solid #e5e5e5;
          border-radius: 3px;

          /* Misc */
          opacity: 1;
    }

Shorthand Notation

Do not use shorthand notation if only applying a value to one dimension.

/* Bad example */
.element {
      margin: 0 0 10px;
      background: red;
      background: url("image.jpg");
      border-radius: 3px 3px 0 0;
}

/* Good example */
.element {
      margin-bottom: 10px;
      background-color: red;
      background-image: url("image.jpg");
      border-top-left-radius: 3px;
      border-top-right-radius: 3px;
}

Advanced CSS Selectors

X Y is descendent selector. Targets the specific element of Y inside the parent of X

X + Y is the adjacent selector. It will select only the element that is immediately preceded by the former element. In this case, only the first paragraph after each ul will have red text.

ul + p {
       color: red;
}

X > Y will only select DIRECT children

X ~ Y sibling combinator is like adjacent selector but is not as specific. While an adjacent selector (ul + p) will only select the first element that is immediately preceded by the former selector, this one is more generalized. It will select, referring to our example above, any p elements, as long as they follow a ul.

ul ~ p {
       color: red;
}

Before and After Elements

Every HTML element has before and after elements that you can style to add content to and style. These are especially useful when working with text or css shapes. Utilizing these tools, you can avoid the use of images, and build directly with code.

Using the content CSS rule, you can inject text, or an object before or after an object. 

// HTML
<p>This is some text</p>
// CSS
p { color:#000; }
p:before {
    content: 'Hello, ';        
}
//Will output
Hello, This is some text

Media Queries

Media queries are used to change the presentational content by limiting the scope of the stylesheet to a certain context set by width, height and color.

The usage of media queries is widespread in responsive design because of the ability to target very specific screen sizes and devices.

A media query can be used multiple ways

Link

<link href="responsive.css" media="screen and (min-width: 701px) and (max-width: 900px)">

In a stylesheet

@media only screen and (min-width: 701px) and (max-width: 900px) {
}

Operators in a media query

Media queries except operators to create complex media queries. These operators and and, only and not

and

and is used to combine media features together into one chain. All features of the media query that you want to select an object must have an and separating them.

Example

@media screen and (min-width: 700px) and (orientation: landscape) { … }

only

only is used to be provide specific styles to certain users. This can be used to prevent certain objects that a browser will not support from displaying.

Example

@media only print { … }

not

not is used to negate all previous queries that include the same context.

Example

@media print not screen { … }

comma-seperated lists

Commas act like or operators between media features.

Example

@media (min-width: 700px), handheld and (orientation: landscape) { … }
The above media query would target devices devices that have a screen larger than 700px and handheld devices that are ONLY in landscape mode. If the handheld device was in portrait mode, the media query would not apply unless it had a screen larger than 700px.

Media Types

Targets the output of the display. Used to target phones, printers, screens, etc.

all Used for all media type devices

aural Used for speech and sound synthesizers

braille Used for braille tactile feedback devices

embossed Used for paged braille printers

handheld Used for small or handheld devices

print Used for printers

projection Used for projected presentations, like slides

screen Used for computer screens

tty Used for media using a fixed-pitch character grid, like teletypes and terminals

tv Used for television-type devices

Media Features

Media features are use to provide the context for the media query. This allows you to target your devices for specific screen types. Some of the most commonly used media features are below

Height

Describes the height of the viewport on a rendering device. Can be used with max and min.

@media (height:800px) { … }
@media (max-height:800px) { … }
@media (min-height:800px) { … }
Width

Describes the width of the viewport on a rendering device

@media (width:1080px) { … }
@media (max-width:1080px) { … }
@media (min-width:1080px) { … }
Orientation

Describes whether the device is in landscape or portrait mode. Very useful for tablet and mobile phones.

@media (orientation: portrait) { … }
@media (orientation: landscape) { … }
Device Pixel Ratio

Used to target higher resolution screens on devices. Mainly retina displays on Apple products.

@media (min-device-pixel-ratio : 1.5) { … }

=====

Chaining these together, we can provide media queries for most devices

Smartphones(Portrait and Landscape)
@media screen and handheld and (min-device-width : 320px) and (max-device-width : 480px) { … }
Retina Devices
@media only screen and (-webkit-min-device-pixel-ratio : 1.5), only screen and (min-device-pixel-ratio : 1.5) { … }
Larger Screens
@media only screen and (min-width : 1820px) { … }
Printers
@media only print { … }

Typography in CSS

Over the years, CSS has provided developers the ability to match a design very closely. Be it drop shadows, kerning, line-height and more, these are all attainable within CSS. In the past, to match these designs, an image was created and inserted in the HTML. This created sites with less accessible elements, and longer load times.

Thankfully, CSS has developed into a strong tool that allows us to match these designs through code.

Basic CSS Typography Styling

  • Font-Size: Allows us to set the pixel size of a font.
  • Font Family: Sets the font for the text to use.
  • Color: Sets the color of a text element through Hex Code, RGBA or others
  • Line-Height: Sets the height of lines of text using pixels.
  • Font-Weight: Sets the weight of text
    • Strong: Bold styling
    • Stronger Bolder Styling
    • Normal: Defines normal, standard characters
    • Light: Light Styling
    • Lighter: Lighter Styling

Advanced CSS Typography Styling

Text Shadow:

Defines a shadow on a text element, much like outer glow and drop shadow in Photoshop.

p {
    text-shadow: 1px 1px 5px #000;
}

The syntax is read as the following:

text-shadow: horizontal-shadow vertical-shadow blur color;

Letter Spacing

Defines the kerning between characters of text, much like in Photoshop

p {
    letter-spacing: 2px;
}

Word Wrap

Long words that extends past the boundaries of a container can be broken automatically by using word-wrap

p {
    word-wrap: break-word;
}

Text-Overflow

Blocks of text that extend past the boundaries of a containing object can be shortened using the text-overflow rule. The options include shortening the text completely by clipping the extended text, or inserting an ellipsis to show that the text has been clipped and there is more present.

p {
    text-overflow: ellipsis;
}

Media Queries

One of the main concepts in responsive development is the use of media queries to define styling at different screen sizes.

The basic media query requires understanding the use of nesting. All styles that are to be defined for different screen sizes are nested within a media query.

How to make a Media Query

The first, most important piece of a media query is stating @media. This tells the browser to look for any styles contained with the media query.

Once you have stated @media you can define a media type or any attributes of the media query required. Using type and an attribute, you can define exactly which device, screen size or a combination of both to target. Within this media query, you can define styles to be changed within.

A basic media query to target desktops on computers:
@media only screen and (min-width: 1200px) {
/ styles go here /
}

The above media query is designed as such: the only screen element states that only devices with screens will be targeted. These include desktops, laptops, smartphones and tablets.

The second portion of the media query - min-width: 1200px is used to define what requirements must be met to display the contents of the media query.

A media query is best used to define new styles for devices other than a desktop. Start designing for desktop, and use media queries to target others within.

Github

When working within a team, keeping code up to date within all members of a team is important. As well, having a central place to keep code protected and showcased is imperative.

Github is an online service to allow code to be managed and showcased. The system it uses is called Git, which was developed by Linus Torvalds, the creator of the operating system Linux in 2005.

Terminology

Profile

Your user page. Displays your repositories and activities on the site.

Repository

Most basically, a project. Think of it as a folder on your computer that contains your code for a project. The repository contains all the project files and keeps the history of all the files.

Commit

A marker of file changes. Essentially a record of an individual save that includes the time, date and who made what changes. Commits contain a message that states what changes were made.

Branch

A version of the repository that contains seperate code. Typically used to add features which will not affect the ‘master’ branch (the main branch) and can be merged back into the ‘master’ branch when completed.

Push

Sends your committed changes from your computer (or local machine) to the repository. Once your commits are pushed to the repository, another user can pull the changes to their computer.

Pull

Downloading the latest changes to the repository to your computer.

Clone

A copy of the files included in the repository or the act of obtaining those files.

Floats

Float is a widely used CSS positioning property. They are designed to be used on images in blocks to text, to allow text to wrap around images, much like in print design. Applying a float in a webpage tells content to flow around the object.

Beyond images, the float property can be used to layout objects on a page.
You can define float: left; and float: right; to tell an object to position itself accordingly.

Think of HTML elements as 100% width blocks. By default they will fill the screen horizontally. With floats and a width, you can tell an element to fill next to each other.

Clearing A Float

The sister element to the Float property is Clear. Using clear, you can tell an object to again respect the block element structure. Using clear:both; on an element will stop the object from breaking the natural flow of a webpage.

Colours

Modern browser can benefit from the use of a broad spectrum of colours to style it’s elements. Typically colours are defined through hexidecimal values, commonly known as ‘hex values’.

Hexidecimal

color: #000000;

Hex Values are read by a browser as a code to colours. It is treated as Red/Green/Blue spectrum with each colour given a two digit group of characters. The first and secret characters represent the red value, the third and forth characters represent the green value, the fifth and sixth characters represent the blue value. The characters range from a-f and 0-9 to create colour combinations.

// Red
color: #ff0000;
// Green
color: #00ff00;
// Blue
color: #0000ff;

RGBA

color: rgba(255,255,255,0.8);

While hex values are great for creating solid colours, what if you need a colour that has some transparency. Enter RGBA, meaning Red Blue Green Alpha. Instead of using hex codes for each value, the colours are defined by a range from 0-255. The final decimal value indicates the alpha level of the colour.

// Red 50% Alpha
color: rgba(255,0,0,0.5);
// Green 20% Alpha
color: rgba(0,255,0,0.2);
// Blue 95% Alpha
color: rgba(0,0,255,0.95);

Backgrounds

Instead of using colours for backgrounds, images can be incorporated in a number of ways in your CSS

Background-Image

Using the background image value can allow an image to be set as your background.

background-image: url('background.jpg');

By default, this image will be the size of the image, will repeat across the page, start in the top left corner, and will scroll with the page.

Background-Repeat

If you do not want the background to repeat across the page, or only want the background to repeat a certain direction or not repeat at all, you can use the background-repeat property.

// Only repeat horizontally
background-repeat: repeat-x;
// Only repeat vertically
background-repeat: repeat-y
// Do not repeat
background-repeat: no-repeat;

Background-Position

You can adjust where the background will begin on your site using the background-position property. This is particularly useful in use with the background-repeat property, if you want an image certained in the page. The values that it takes, work on the x and y dimensions of a page or by simply defining generic positioning values.

// Center Background Image vertically and horizontally
background-position: center center;
// Background Image at top left corner
background-position: left top;
//Background Image 200px from left and 500px from top
background-position: 200px 500px;
//Background Image 150px from left and at the bottom
background-position: 150px bottom;

Background-Attachment

By default, all background images will scroll with the page. You can change this functionality by defining a fixed value for the background-attachment property.

//Fixed Position Background
background-attachment: fixed;

Background Size

By default, background images will use the size that they are saved at, but you can define the size that they are presented on a page. These images can be defined using width and height values in pixels or percentages.

// Background Size 800px wide and 200px tall
background-size: 800px 200px;
// Background Size 100% wide and 300px tall
background-size: 100% 300px;
// Background full size of screen
background-size: cover;

Box Model

One of the most misunderstood things about a web page is how an object on the object reacts to dimensions given to it. Using border and padding will affect the dimensions of the object, often in ways, that are undesirable.

Margin is used to create an area around the object. This will affect the layout and positioning of objects around it.

Borders are used to define a border around an object. The width of the border, will affect the positioning and layout of objects around it.

Padding is used to create spacing around the content in an object. The value of the padding will affect the position and layout of objects around it.

Fixing the box model problem.

Padding is one of these things that was not developed well within CSS. Padding should not affect the width and height of the object and not move the content around it. Instead, it should give spacing inside the object.

To fix this problem, the box-sizing property can be used to have padding be used in a more effective way.

Use the following code in your CSS to fix the box-model. The asterisk is used to apply it to all objects on the page.

* {
    box-sizing: border-box;
}

Fixed Vs Fluid Layout

Typically, webpages use pixel values to define the dimensions of an object. While this is great for building a solid page, this can make building a page for different devices, very difficult.

Enter flexible layouts.

Using percentage values can ensure your page reacts to the size of the browser. On smaller screens, fixed pixel values will make an object be too large. Percentage based values are based on 0%-100% and relative to it’s encapsulating object. Using a max-width property on the container object will allow the object to have a flexible width.

Fixed Layout with full header, section and aside next to each other

// HTML
<div id="container">
    <header>Header Content</header>
    <section>Section Content</section>
    <aside>Aside Content</aside>
</div>
// CSS
#container {
    width: 1080px;
    margin: 0 auto;
}
header {
    width: 1080px;
}
section {
    float: left;
    width: 740px;
}
aside {
    float: left;
    width: 340px;
}

Styling is a bad thing

By default, a site without styling is a perfect website. By adding CSS, we are adding extra weight to our users and need to be sure that we give a consistent experience across all browsers.

Reset Your Browser

As you may have noticed, there are a world of browsers out there. Each browser has their own intricacies that may affect your design, such as margins, padding and typography.

Using a reset stylesheet, you can have a clean starting point for each browser. These reset stylesheets enables a browser like Chrome have many of the same styling profile as Firefox.

Semantic Elements

Before HTML5 was in the picture, HTML elements could be defined with a Div with an ID or a class to be used to styling it. This can lead to confusion on a page as it will be difficult to tell what element does what in the code. Semantics are used to define exactly what role the content plays on a website.

Structural Elements

Div - A generic container with special meaning

Form - A container that holds form elements on a page

Table - Contains tabular data

  • tbody - A block of rows that contains the data of a table
  • thead - A block of rows that represent the labels of a table
  • tfoot - A block of rows that represent the summaries or totals of a table
  • tr - A row of cells in a table. The horizontal row of a table
  • td - A data cell of a table. Contained in a row

Img - Defines an image on a page. Requires a source path to an image.

Typography Elements

Typography Elements include

Header Typography -
Used for titles and emphasis

<h1>Title 1</h1>
<h2>Title 2</h2>
<h3>Title 3</h3>
<h4>Title 4</h4>
<h5>Title 5</h5>
<h6>Title 6</h6>

Paragraph

<p>content goes here</p>

Links

<a href="link path goes here">Link Text</a>

Lists

  • List Element - li

      <li>List Element</li>

  • Unordered List - ul

    • Element 1
    • Element 2
    • Element 3
        <ul>
      <li>Element 1</li>

  </ul>

HTML5 Elements

HTML5 introduced new elements that were descriptive as to what part of the page they represented. Instead of using div id=”header”, you can use header along with a class or id for specific styling.

New HTML5 Elements include

Header - Defines the header of a page or a section. Contains the title of a page or a section.

<header>
    <h1>Page Title!</h1>
</header>

Footer - Defines the footer of a page or a section

<footer>
    Copyright 2014 - Drew Minns
</footer>

Nav - Defines a section that contains only Navigation elements

<nav>
    <a href="home.html">Home</a>
    <a href="page.html">Page</a>
</nav>

Section - Defines a section in a document. Use an ID or a class to add styling and specificity.

<section id="post">
    Content Goes Here
</section>

Article - Defines an individual section of content. Typically used for a post in a blog or a collection of content.

<article>
    Vestibulum id ligula porta felis euismod semper. Donec sed odio dui.
</article>

Aside - Defines a section of content that is secondary to the main content. Often used as a sidebar.

<aside>
    Vestibulum id ligula porta felis euismod semper. Donec sed odio dui.
</aside>

Form Elements

All forms need to be contained within form tags. These allows the form elements to be found easily within a page. These form elements can be styled through CSS

Form elements that can be used include

Label - Used to define a label for a input field.

<label>First Name <input></label>



Input - Fields that are used to input information into a page. Specific types of information can be defined through the type attribute on an input

  • type=”text” - Any type of content can be entered

      <label>Your Name <input type="text"></label>



  • type=”number” - Only numbers may be entered

      <label>Phone Number <input type="number"></label>



  • type=”email” - Only email address can be entered

      <label>Email Address <input type="email"></label>



  • type=”tel” - Used for Phone Numbers

      <label> Telephone Number <input type="tel"></label>



  • type=”file” - Used for file uploads

      <label>Select a File <input type="file"></label>



  • type=”radio” - Used to provide radio buttons on a form. Use the name attributes need to be added to group the elements together. The checked attribute is used to define a default checked option. The value attribute is used to define the selected value.

      <label><input type="radio" name="selected" value="Option 1" checked="checked">Option 1</label>
  <label><input type="radio" name="selected" value="Option 2">Option 2</label>



  • type=”checkbox” - Used to define checkboxes. Attributes are similar to radio buttons: checked, values, name.

      <label><input type="checkbox" value="Option 1"> Option 1</label>
  <label><input type="checkbox" value="Option 2"> Option 2</label>



  • type=”submit” - Used to submit values in a form. The value attribute can be used to change the name of the text displayed.

    <input type="submit" value="Send">
