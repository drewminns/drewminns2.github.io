Week 10 - jQuery Functions and Plugins

Download Exercise Files

So we walked through jQuery last class, and discussed how to include it in your page, how to use event handlers to ‘listen’ for clicks, hovers and other actions. As well, we covered inserting text and HTML into objects on the webpage. A quick review:

If we need a fuller review, see my last post

Including jQuery

Include jQuery in the head of your page or before the closing body tag either by using a CDN or including it directly in the folder structure of your site.

 // From a CDN
 <script src="http://code.jquery.com/jquery-1.11.0.min.js"></script>

 // Directly in the project folder
 <script src="js/jquery-1.11.0.min.js"></script>

Where to write your jQuery

Create a new JS file and write your jQuery there. Be sure to link it AFTER you call the jQuery library above. Be sure to include your document ready call in order to load all scripts after the entire page is loaded. The $ is same as writing jquery. It calls the library.

 // Your jQuery file
 <script src="js/main.js"></script>

 // Main.js
 $(document).ready(function() {
      // Write your jQuery in here
 });

Selecting Objects

In order to select an html element on your page with jQuery, you can simply write $(‘objectname’).

Event Handlers

Providing interaction with objects is quite simple, you can state how an object is going to be interacted with and then the result. The result can be the object that was interacted with does something, or another element does.

 //$(document).ready(function() {

      $('#div').click(function() {
           // Do Something
      });

 });

Functions and Plugins

This class is an in class exercise where the details will be discussed in further than in the notes. But we will be extending what we learned last week about selecting objects, event handlers and animations to create a lightbox tool for a gallery.

Creating a function

A function is an encapsulated, generic tool that provides some action, and can be reused whereever necessary. Much like an effect that we call after a selector, we can provide our own name for the function and call it when needed.

 //Create a function and name it hello
 $.fn.hello = function () {

      //Give it some action and apply it to 'this'
      //In this case, changing the css of an object to have green text
      this.css("color", "green");

      //Return the change
      return this

 };

 //Call the function on an object
 $('#div').hello();

In the above example, we created a function called hello(); and then applied to #div. Any text inside will be affected by the action we set in the function, which is changing the color of the text to green, via CSS.

Creating a plugin

A plugin is a function but often with other settings that you can call. More often than not, you call a plugin on an object, the exact same way that you call a function. In order to provide some flexibility to the user, you can set options that can use.

 // hello.js
 // Create a function and name it goodbye
 // This plugin can go in a seperate file, be sure to link to the file on your main page, just like you did with jQuery
 $.fn.hello = function (options) {

      // Define default settings for the plugin
      // Here we are saying, black text and line height of 32px
      var settings = $.extend({
           color: "#000",
           lineHeight: "32px"
      }, options);

      // Return the changes along with the default settings
      return this.css({
           color: settings.color,
           lineHeight: settings.lineHeight
      });
 }

 // main.js
 $(document).ready(function() {

      // Call the plugin and use the settings to input your own values
      $('#div').hello({
           color: "#00ff00",
           lineHeight: "48px"
      });

 });

Week 9 - jQuery

jQuery

Download In-Class Files https://github.com/drewminns/webd-starter

Ok, first things first.

Javascript is a programming language. jQuery is a library that uses Javascript to make programming user actions on a webpage easier. jQuery was developed to remove some of the more strenuous activities that come with selecting objects on a webpage and interacting, animating and more.

DOM

The Document Object Model is the framework of all elements on a webpage. This includes all objects and elements we place on the page. The web inspector in your browser is the best way to explore the DOM.

CDN VS Hosted

In order to use jQuery, we must import the library into our page. Because Javascript is built into every browser, we simply import the library via a script tag, that hooks into the existing Javascript system.

http://jquery.com/ hosts all the documentation needed and allows you to download versions of jQuery to include in your site heirarchy.

Another option of including jQuery in your page, is using a CDN (Content Delivery Network) to serve the file to your browser. Using a CDN ensures that even if you forget to upload jQuery, it will be linked to your site.

<script src="http://code.jquery.com/jquery-1.11.0.min.js"></script>

Version 1 vs 2.0

jQuery 2.0 was recently released and is the next logical iteration of jQuery to use. While we do our best in this industry to stay up to date, 2.0 does not support IE 6/7/8, which means that we would be leaving out a chunk of our users. That’s why it is best to currently stick with a 1.0 version, particularly 1.11.

Document Ready

To ensure that our page has all functionality for a user to use, we have to ensure that the page is entirely loaded before applying any jQuery animations. To do that, we use a function to check that the DOM is ready. Write this function once and insert all jQuery inside.

$(document).ready(function() {
    // Put jQuery in here
});

Without ensuring that the document is loaded, we risk our javascript not being loaded and functionality being lost. It is imperative that we use the above script at all times when using jQuery.

One of the easiest ways to check that jQuery is working is to put a console.log(“It works!”); inside of the document ready function.

$(document).ready(function() {
    console.log("It works!");
});

With this, we can then use the console in our browser inspector to view the message.

Selecting Objects

In vanilla Javascript, selecting an object was a convuluted task.

//HTML
<div id="myObject"></div>
//JS
var x=document.getElementById("myObject");

From there, we would have to use the variable x to apply animations. With jQuery we use the character $ to instantiate a jQuery command and selecting an object.

//HTML
<div id="myObject"></div>
$(document).ready(function() {
    $("#myObject").someFunction();
});

Functions

jQuery has built in functions that we can utilize to make changes. They are all listed within the API documentation at http://api.jquery.com/. They can be changed together and applied to an object as often as wanted.

$("h3").slideToggle().css("height", "300px").hide();

Changing CSS

Once we have an object selected we can easily apply functions to it that are apart of the jQuery library.

One of those handy functions is css. Using the css function, we can easily change the CSS attributes of an object.

$("h1").css("color", "#fff");

Because the jQuery is applied after the document is loaded, the changes will overwrite anything within the stylesheet.

Adding Classes

Another great usage of jQuery is adding classes to objects, which then have styles applied to them.

//HTML
<div id="box"></div>

// JS
$("#box").addClass("blueBox");
//Outputs
<div id="box" class="blueBox"></div>

Inserting text or HTML to an object

Another group of functions that can be utilized is text() and html(). Using these functions, you can insert content inside of a selected object. With text(), you can insert simple plain text into an object. With the html() function, you can insert html inside of the selected element.

//HTML
<div id="one"></div>
<div id="two"></div>
//JS
$("#one").text("New Content");
$("#two").html("<h2>Look at me!</h2>");
//Outputs
<div id="one">New Content</div>
<div id="two"><h2>Look at me!</h2></div>

Allowing with inserting content inside of an object, we can use .prepend() and .append() to specify where in the object we want to insert the content.

append();

//HTML
<div id="one">
    Hello
</div>
<div id="two">
    Hello
</div>
//JS
$("#one").append("<p>Goodbye</p>");
//Output
<div id="one">
    Hello
    <p>Goodbye</p>
</div>
<div id="two">
    Hello
</div>

prepend();

//HTML
<div id="one">
    Hello
</div>
<div id="two">
    Hello
</div>
//JS
$("#one").append("<p>Goodbye</p>");
$("#two").prepend("<p>Goodbye</p>");
//Output
<div id="one">
    Hello
    <p>Goodbye</p>
</div>
<div id="two">
    <p>Goodbye</p>
    Hello
</div>

Event Handlers

jQuery comes with a wide array of tools used to interact with an object. These handlers bind to the Javascript events that occur in a browser.

Events pertaining to a users mouse actions include click, hover, mousedown, mouseup, mouseleave and more. For the full list, check out http://api.jquery.com/category/events/mouse-events/

For events pertaining to keyboard actions, read http://api.jquery.com/category/events/keyboard-events/

There are many more that can be utilized with the browser itself, a form or a document loading. http://api.jquery.com/category/events/

Every action needs a reaction

For every event handler used, you typically would need to include some to happen. For example, click an object and turn another object green. To do this, we select an object, attach an event handler to it, instatiate a function and insert the reaction within. See below

//CSS
#blue {
    color: #0000ff;
}
//HTML
<h1>Click</h1>
<div id="blue">Hello</div>
//JS
// Select object and apply event handler
$("h1").click(function () {
    // Apply change to another object
    $("#blue").css("color", "#00ff00");
});

The above example would work like this. When the h1 element is clicked, the div with the ID of blue, would have it’s text colour changed to green (#00ff00).

Apply to itself

Now, what if we didn’t want the click to apply a change to another element, we wanted the click to change itself. We can simply use this as the selector.

//HTML
<h1>Click</h1>
<div id="blue">Hello</div>
//JS
// Select object and apply event handler
$("h1").click(function () {
    // Apply change to another object
    $(this).css("color", "#00ff00");
});

Note that this is not inside quotes, because it is a root object. The output would be that the h1 would change colour once it is clicked.

Making our own functions

We can write our own functions to reuse throughout a page, much like how functions are created in regular Javascript. Be sure to include return this to return the change. The use of created functions are imperative to not repeating code over and over. Write it once, and call it as many times as needed.

// Instantiate a function and name it greenify
$.fn.greenify = function() {
  // Apply CSS changes
  this.css( "color", "green" );
  // Return the changes
  return this;
};
// Call greenify function on object
$("h1").greenify();

Animation

Yes, animation. The coolest part of it all. Goodbye Flash, hello Javascript. The way it should be. Animations fall in to the category of Effects in jQuery http://api.jquery.com/category/effects/. With animations, you can fade objects, hide, show, slide, etc. These work by manipulating the dimensions and opacity of HTML elements. They are best used in conjunction with an event handler.

// Using the generic animate effect, we can pass a css rule in to be manipulated on an event. This being on hover, we will change the font size to 48px
$("h1#block").hover(function() {
    $(this).animate({ "font-size":"48px"}, 1000);
}

Oh, that number at the end? That’s the duration. It determines how long the animation will run. A number can be passed through that represents milliseconds. You can also use strings such as Fast and Slow to represent 200 and 600 milliseconds respectively.

Week 8 - Best Practices

Best Practices or: Why we do the things we do

So websites are built with code and images, and ultimately your code could look however you want it to, and your website would still work. However, what happens when you go back to edit your code and there’s no rhyme or reason to how it looks. Now, think of yourself as someone who’s working on a project that someone else started. That person left their code a mess, and it’s not part of your job, and quoted hours, cleaning it up.

It’s important that we maintain standards within our code, not only to help ourselves and others, but there are significant performance and debugging issues when we don’t follow standards.

Below are some common standards followed throughout the Industry

HTML

Syntax

  • Use tabs instead of spaces to format your code. Specifically, use 2 tabs.
    • In Sublime, go to View -> Indentation and set Tab-Width to 2 and always use tabs.
  • Indent nested elements by using tabs

  • Use double quotes on your attributes

      <!DOCTYPE html>
  <html>
    <head>
      <title>Page title</title>
    </head>
    <body>
      <img src="images/company-logo.png" alt="Company">
      <h1 class="hello-world">Hello, world!</h1>
    </body>
  </html>

Practicality

Use the least amount of markup at all times while maintaning simplicity. Avoid using unnecesary elements that will require extra CSS

<!-- Bad -->
<div class="profile">
    <img src="#">
</div>

<!-- Best -->
<img src="#" class="profile">

Attribute Order

Order your attributes accordingly to help with debugging your code.

  • class
  • id
  • name
  • data-*
  • src, for, type, href
  • title, alt

CSS

Syntax

  • Use tabs instead of spacing
  • When grouping selectors, keep each selector to a single line
  • Include one space before the opening curly brace
  • Put closing brackets on a new line
  • Include one space after each colon for each declaration
  • Each CSS rule should appear on it’s own line
  • Lowercase each hex value
  • Use shorthand values where necessary
  • Indent the CSS rules
    /* Bad */
    .box1, .box2, .box3, .box4{
    color:#000000; background-color:#ffffff;}
    /* Good */
    .box1,
    .box2,
    .box3,
    .box4 {
        color: #000000;
        background-color: #ffffff;
    }

Declaration Order

CSS Rules should be declared in a certain order in order to not group certain aspects together

  • Position
  • Box Model
  • Typographic
  • Visual
    .declaration-order {
          /* Positioning */
          position: absolute;
          top: 0;
          right: 0;
          bottom: 0;
          left: 0;
          z-index: 100;

          /* Box-model */
          display: block;
          float: right;
          width: 100px;
          height: 100px;

          /* Typography */
          font: normal 13px "Helvetica Neue", sans-serif;
          line-height: 1.5;
          color: #333;
          text-align: center;

          /* Visual */
          background-color: #f5f5f5;
          border: 1px solid #e5e5e5;
          border-radius: 3px;

          /* Misc */
          opacity: 1;
    }

Shorthand Notation

Do not use shorthand notation if only applying a value to one dimension.

/* Bad example */
.element {
      margin: 0 0 10px;
      background: red;
      background: url("image.jpg");
      border-radius: 3px 3px 0 0;
}

/* Good example */
.element {
      margin-bottom: 10px;
      background-color: red;
      background-image: url("image.jpg");
      border-top-left-radius: 3px;
      border-top-right-radius: 3px;
}

Advanced CSS Selectors

X Y is descendent selector. Targets the specific element of Y inside the parent of X

X + Y is the adjacent selector. It will select only the element that is immediately preceded by the former element. In this case, only the first paragraph after each ul will have red text.

ul + p {
       color: red;
}

X > Y will only select DIRECT children

X ~ Y sibling combinator is like adjacent selector but is not as specific. While an adjacent selector (ul + p) will only select the first element that is immediately preceded by the former selector, this one is more generalized. It will select, referring to our example above, any p elements, as long as they follow a ul.

ul ~ p {
       color: red;
}

Before and After Elements

Every HTML element has before and after elements that you can style to add content to and style. These are especially useful when working with text or css shapes. Utilizing these tools, you can avoid the use of images, and build directly with code.

Using the content CSS rule, you can inject text, or an object before or after an object. 

// HTML
<p>This is some text</p>
// CSS
p { color:#000; }
p:before {
    content: 'Hello, ';        
}
//Will output
Hello, This is some text
