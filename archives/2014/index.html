Styling is a bad thing

By default, a site without styling is a perfect website. By adding CSS, we are adding extra weight to our users and need to be sure that we give a consistent experience across all browsers.

Reset Your Browser

As you may have noticed, there are a world of browsers out there. Each browser has their own intricacies that may affect your design, such as margins, padding and typography.

Using a reset stylesheet, you can have a clean starting point for each browser. These reset stylesheets enables a browser like Chrome have many of the same styling profile as Firefox.

    -moz-box-sizing: content-box;

Semantic Elements

Before HTML5 was in the picture, HTML elements could be defined with a Div with an ID or a class to be used to styling it. This can lead to confusion on a page as it will be difficult to tell what element does what in the code. Semantics are used to define exactly what role the content plays on a website.

Structural Elements

Div - A generic container with special meaning

Form - A container that holds form elements on a page

Table - Contains tabular data

  • tbody - A block of rows that contains the data of a table
  • thead - A block of rows that represent the labels of a table
  • tfoot - A block of rows that represent the summaries or totals of a table
  • tr - A row of cells in a table. The horizontal row of a table
  • td - A data cell of a table. Contained in a row

Img - Defines an image on a page. Requires a source path to an image.

HTML5 Elements

HTML5 introduced new elements that were descriptive as to what part of the page they represented. Instead of using div id=”header”, you can use header along with a class or id for specific styling.

New HTML5 Elements include

Header - Defines the header of a page or a section. Contains the title of a page or a section.

<header>
    <h1>Page Title!</h1>
</header>

Footer - Defines the footer of a page or a section

<footer>
    Copyright 2014 - Drew Minns
</footer>

Nav - Defines a section that contains only Navigation elements

<nav>
    <a href="home.html">Home</a>
    <a href="page.html">Page</a>
</nav>

Section - Defines a section in a document. Use an ID or a class to add styling and specificity.

<section id="post">
    Content Goes Here
</section>

Article - Defines an individual section of content. Typically used for a post in a blog or a collection of content.

<article>
    Vestibulum id ligula porta felis euismod semper. Donec sed odio dui.
</article>

Aside - Defines a section of content that is secondary to the main content. Often used as a sidebar.

<aside>
    Vestibulum id ligula porta felis euismod semper. Donec sed odio dui.
</aside>

Form Elements

All forms need to be contained within form tags. These allows the form elements to be found easily within a page. These form elements can be styled through CSS

Form elements that can be used include

Label - Used to define a label for a input field.

<label>First Name <input></label>



Input - Fields that are used to input information into a page. Specific types of information can be defined through the type attribute on an input

  • type=”text” - Any type of content can be entered

      <label>Your Name <input type="text"></label>



  • type=”number” - Only numbers may be entered

      <label>Phone Number <input type="number"></label>



  • type=”email” - Only email address can be entered

      <label>Email Address <input type="email"></label>



  • type=”tel” - Used for Phone Numbers

      <label> Telephone Number <input type="tel"></label>



  • type=”file” - Used for file uploads

      <label>Select a File <input type="file"></label>



  • type=”radio” - Used to provide radio buttons on a form. Use the name attributes need to be added to group the elements together. The checked attribute is used to define a default checked option. The value attribute is used to define the selected value.

      <label><input type="radio" name="selected" value="Option 1" checked="checked">Option 1</label>
  <label><input type="radio" name="selected" value="Option 2">Option 2</label>



  • type=”checkbox” - Used to define checkboxes. Attributes are similar to radio buttons: checked, values, name.

      <label><input type="checkbox" value="Option 1"> Option 1</label>
  <label><input type="checkbox" value="Option 2"> Option 2</label>



  • type=”submit” - Used to submit values in a form. The value attribute can be used to change the name of the text displayed.

    <input type="submit" value="Send">
