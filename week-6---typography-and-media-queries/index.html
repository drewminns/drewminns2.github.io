Typography in CSS

Over the years, CSS has provided developers the ability to match a design very closely. Be it drop shadows, kerning, line-height and more, these are all attainable within CSS. In the past, to match these designs, an image was created and inserted in the HTML. This created sites with less accessible elements, and longer load times.

Thankfully, CSS has developed into a strong tool that allows us to match these designs through code.

Basic CSS Typography Styling

  • Font-Size: Allows us to set the pixel size of a font.
  • Font Family: Sets the font for the text to use.
  • Color: Sets the color of a text element through Hex Code, RGBA or others
  • Line-Height: Sets the height of lines of text using pixels.
  • Font-Weight: Sets the weight of text
    • Strong: Bold styling
    • Stronger Bolder Styling
    • Normal: Defines normal, standard characters
    • Light: Light Styling
    • Lighter: Lighter Styling

Advanced CSS Typography Styling

Text Shadow:

Defines a shadow on a text element, much like outer glow and drop shadow in Photoshop.

p {
    text-shadow: 1px 1px 5px #000;
}

The syntax is read as the following:

text-shadow: horizontal-shadow vertical-shadow blur color;

Letter Spacing

Defines the kerning between characters of text, much like in Photoshop

p {
    letter-spacing: 2px;
}

Word Wrap

Long words that extends past the boundaries of a container can be broken automatically by using word-wrap

p {
    word-wrap: break-word;
}

Text-Overflow

Blocks of text that extend past the boundaries of a containing object can be shortened using the text-overflow rule. The options include shortening the text completely by clipping the extended text, or inserting an ellipsis to show that the text has been clipped and there is more present.

p {
    text-overflow: ellipsis;
}

Media Queries

One of the main concepts in responsive development is the use of media queries to define styling at different screen sizes.

The basic media query requires understanding the use of nesting. All styles that are to be defined for different screen sizes are nested within a media query.

How to make a Media Query

The first, most important piece of a media query is stating @media. This tells the browser to look for any styles contained with the media query.

Once you have stated @media you can define a media type or any attributes of the media query required. Using type and an attribute, you can define exactly which device, screen size or a combination of both to target. Within this media query, you can define styles to be changed within.

A basic media query to target desktops on computers:
@media only screen and (min-width: 1200px) {
/ styles go here /
}

The above media query is designed as such: the only screen element states that only devices with screens will be targeted. These include desktops, laptops, smartphones and tablets.

The second portion of the media query - min-width: 1200px is used to define what requirements must be met to display the contents of the media query.

A media query is best used to define new styles for devices other than a desktop. Start designing for desktop, and use media queries to target others within.
